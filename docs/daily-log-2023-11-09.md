## Decisiones de diseño
1. **Estructura de paneles**: Optamos por una disposición de tres secciones principales 
   (encabezado, cuerpo, pie) con el cuerpo dividido en dos paneles (errores y comandos) 
   para maximizar el espacio de visualización.
   
2. **Biblioteca Rich**: Seleccionamos Rich para la implementación de TUI debido a sus 
   capacidades robustas para crear interfaces en terminal con mínimo esfuerzo y máxima 
   compatibilidad entre plataformas.

3. **Modo de demostración**: Implementamos un modo demo separado para facilitar las 
   pruebas y presentaciones sin necesidad de repositorios de prueba específicos.
